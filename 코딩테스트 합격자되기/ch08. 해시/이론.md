# 해시의 개념
- 해시 함수를 사용해서 변환한 값을 인덱스로 삼아, 키와 값을 저장해서 빠른 데이터 탐색을 제공하는 자료구조
  - 키: 값을 찾기 위한 정보
  - 값: 찾으려는 정보

## 해시(테이블)의 특징
- 해시함수는 단방향으로 동작
- 탐색시간이 O(1)로 짧음
- 값을 인덱스로 사용하려면 적절한 변환 과정 필요
- ![](https://i.sstatic.net/0yjYd.png)

## 해시의 특성을 활용하는 분야
- 비밀번호의 관리: 비밀번호를 해시 값으로 저장함
- 데이터베이스 인덱싱: DB에 저장된 데이터를 효율적으로 검색할 때 해시를 활용
- 블록체인: 이전 블록의 해시값을 가지고 있어, 전체 블록에 대한 무결성을 보장

# 해시 함수
## 해시 함수를 구현할 때 고려할 내용
- 해시 값은 인덱스로 활용해야 하므로, 해시 테이블의 크기를 넘으면 안된다
- 해시 값의 충돌은 최대한 적게 발생해야 한다.

## 자주 사용하는 해시 함수 알아보기
### 나눗셈법
- 나머지 값을 활용하는 연산을 통해, 인덱스를 구하는 방법
  - 키를 소수로 나눈 나머지를 인덱스로 사용함
- 소수를 사용하는 이유?
  - 다른 수를 사용할 때 보다 충돌이 적기 때문
  - N배수를 중심으로 해시값이 반복되기 때문
  - 해시테이블의 크기는 소수의 수에 해당

### 곱셈법
- 큰 소수를 사용해야하는 나눗셈법의 단점을 보완한 방법
- 계산 방법
  1. 키에 황금비를 곱한다.
  2. 1단계에서 구한 값에 모듈러 1을 취한다.
  3. 2단계에서 구한 값을 기반으로 실제 해시 테이블에 매핑

### 문자열 해싱
- 문자를 숫자로 치환하고, 이 값을 다항식의 값으로 변환하는 방법
- 계산 방법
  1. 문자를 숫자로 치환한다.
  2. 문자열에 31의 n승을 곱한다.
  3. 2단계의 값을 더한 뒤 해시테이블 최대 크기와 모듈러 연산을 한다.
- 오버플로를 방지하는 방법
  - 2단계에서 해시테이블 최대 크기 만큼을 모듈러 연산을 수행한다.

# 충돌 처리
## 해시 충돌의 해결 방법
![](https://velog.velcdn.com/images/redoyp/post/9fb7291a-aadb-40f3-9b0d-f50a6a93242b/image.png)
### 체이닝
- 충돌이 발생한 데이터를 연결 리스트로 저장하는 방식.

### 개방 주소법
- 충돌이 발생한 데이터를 해시 테이블의 다른 빈 공간에 저장하는 방식. 
  - 선형 조사법: 충돌이 발생할 경우 해시 테이블의 다음 빈 공간을 찾는 방식
    - 군집화: 해시 충돌이 빈번하게 발생하여, 데이터가 특정 구역에 집중되는 현상, 선형 조사법에서 주로 발생
    - 이차 조사법: 충돌이 발생할 경우 일정한 간격으로 떨어진 빈 공간을 찾는 방식
### 이중 해싱
- 두 개의 해시 함수를 사용하여 충돌을 해결하는 방식
