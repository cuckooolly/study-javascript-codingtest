# 집합과 상호배타적 집합의 개념

## 집합의 개념
- 순서와 중복이 없는 원소들을 갖는 자료구조

### 집합의 종류
- 상호배타적 집합: 교집합이 없는 집합 관계
- 그래프 알고리즘에서 상호배타적 집합이 많이 쓰임

## 상호배타적 집합의 특성을 활용하는 분야
- 이미지 분할: 이미지를 서로 다른 부분으로 나누는 데 사용
- 도로 네트워크 구성: 도로를 구축할 때 각 도로를 서로 교차하지 않도록 설계하는데 사용
- 최소 신장 트리 알고리즘 구현: 간선을 추가할 때마다 사이클을 형성하는 지 확인할 때 사용
- 게임 개발: 캐릭터의 동작을 구현. 캐릭터가 겹치지 않게 할 수 있음.
- 클러스터링 작업: 각 작업이 서로 겹치지 않도록 구성하여, 여러 작업을 동시에 수행할 수 있도록 함.

# 집합의 연산

## 배열을 활용한 트리로 집합 표현하기

### 대표원소란?
- 집합의 원소 중 집합을 대표하는 역할을 수행하는 원소
- 여기서는 집합의 형태를 트리로 나타내므로, 루트 노드로써 표현할 예정

### 배열로 집합을 표현하는 것이란?
- 하나의 배열로 상호배타적 관계를 가지는 집합을 모드 표현
- 배열의 인덱스는 자신을, 배열값은 부모노드를 의미
- 배열의 크기: 배열의 인덱스가 모든 집합의 원소를 표현할 수 있으면 됨.

## 유니온-파인드 알고리즘
- 집합 연산에서 자주 쓰이는 합치기와 탐색 연산을 수행하는 알고리즘

### 파인드 연산

- 특정 노드의 루트 노드가 무엇인지 탐색하는 방법
- 특정 노드가 같은 집합에 있는지 확인할 때 사용
- 연산 과정
  1. 현재 노드의 부모 노드를 확인
  2. 부모 노드가 루트 노드이면 찾기 연산을 종료
  3. 찾기 연산이 종료되지 않으면 1번을 반복

### 파인드 연산의 연산 비용 문제, 경로 압축으로 해결하자
![](https://velog.velcdn.com/images/chanmi125/post/eb95b7cf-9cc1-41be-be18-5afe10443a2e/image.png)

- 경로 압축을 통해, 트리의 높이를 줄여 연산의 규모를 줄일 수 있음

### 유니온 연산

- 두 개의 집합을 하나로 합치는 연산
  - 두 집합의 루트노드를 같게 만드는 것
- 연산 과정
  1. 두 집합에서 찾기 연산을 통해 루트 노드를 찾음
    ![](https://velog.velcdn.com/images/chanmi125/post/d9880f1a-09ab-417e-8fcc-ec16b9ce036b/image.png)
  2. 찾은 두개의 루트 노드의 값을 비교
  3. 두 집합을 합침. (한 집합의 루트 노드를 다른 집합의 루트 노드로 변경)
    ![](https://velog.velcdn.com/images/chanmi125/post/7309b3e5-808f-4f4a-8cf7-6c8e2b1c5c21/image.png)

### 유니온 연산의 연산 비용 문제, 랭크로 해결하자

- 랭크: 현재 노드를 기준으로 하였을 때, 가장 깊은 노드까지의 경로 길이
- 랭크 기반으로 합치기 연산하기
  1. 두 노드의 루트 노드를 구함
  ![](https://velog.velcdn.com/images/chanmi125/post/d9880f1a-09ab-417e-8fcc-ec16b9ce036b/image.png)
  2. 1에서 구한 루트 노드의 랭크를 비교
     1. 랭크 값이 다르면, 랭크 값이 큰 루트 노드를 기준으로 삼음
     2. 랭크 값이 같으면, 루트 노드를 아무거나 선택해서 바꾸고, 최종 루트 노드의 랭크에 1을 더함
     ![](https://velog.velcdn.com/images/chanmi125/post/7309b3e5-808f-4f4a-8cf7-6c8e2b1c5c21/image.png)
