# 시뮬레이션 문제 풀이 노하우

## 시뮬레이션 문제를 푸는 방법

- 하나의 문제를 여러 개로 분리한다.
  - 문제를 분리하지 않고 하나의 코드로 구현하면, 코드가 복잡해진다.
- 예외 처리가 필요하다면 독립함수로 구현한다.
  - 기본적으로 동작하는 부분과 예외처리 부분의 코드가 섞이면 디버깅이 힘들어진다.

## 행렬 연산

### 행렬 덧셈과 뺄샘, 그리고 곱셈

- 덧셈/뺄셈: 같은 크기의 행렬끼리 대응하는 원소끼리 더하거나 뺀다.
- 곱셈: 행렬 A의 행과 행렬 B의 열을 곱해서 더한 값을 행렬 C의 원소로 넣는다.
  - 행렬 A의 열 개수와 행렬 B의 행 개수가 같아야 한다.
  - 행렬 A가 m x n 크기이고, 행렬 B가 n x p 크기라면, 행렬 C는 m x p 크기가 된다.

### 전치 행렬

- 행과 열을 바꾼 행렬을 전치 행렬이라고 한다.
- 전치 행렬의 구현은 행과 열을 바꾸면 된다.

## 좌표 연산

### 좌표 배열로 표현하기

- 좌표값에 해당하는 배열의 위치를 활용한다.
  - (3,4)의 위치를 배열 arr[4][3]=1과 같이 표현할 수 있다.

### 좌표 이동 오프셋 값으로 쉽게 표현하기

- 상하좌우, 대각선 이동을 오프셋 배열로 선언한다. 그 후 반복문을 통해 이동한다.
    - 예시) 
        ```javascript
        const dx = [-1, 0, 1, -1, 1, -1, 0, 1];
        const dy = [-1, -1, -1, 0, 0, 1, 1, 1];
      
        for (let dir = 0; dir < 8; dir++) {
            const nx = x + dx[dir];
            const ny = y + dy[dir];
            // (nx, ny)로 이동하는 코드
        }
        ```

## 대칭, 회전 연산
- 대칭 연산의 일반화
    - 좌우 대칭: x좌표는 변하지 않고 y좌표만 변화 (x, y) -> (x, -y)
    - 상하 대칭: y좌표는 변하지 않고 x좌표만 변화 (x, y) -> (-x, y)
    - 대각선 대칭: x좌표와 y좌표가 서로 바뀜 (x, y) -> (y, x)
- 회전 연산
    - 90도 회전: (x, y) -> (y, -x)
    - 180도 회전: (x, y) -> (-x, -y)
    - 270도 회전: (x, y) -> (-y, x)
